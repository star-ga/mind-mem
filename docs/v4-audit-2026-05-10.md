## Action items

### Fixed in this commit (highest leverage, correctness first)

1. **Tier-memory CAS** (closes the unanimous blind spot)
   - Add `block_version INTEGER NOT NULL DEFAULT 0` column to
     `block_recall_tier`.
   - Add `expected_version` parameter to all tier-write functions
     (lands when promotion writes are added).
   - Reject stale writes with `StaleVersionError`.
2. **Multi-label `block_kinds`** (3/4 agreement)
   - Add additive `block_kind_tags(block_id, kind, PRIMARY KEY(block_id, kind))`
     junction table.
   - Single-label callers keep working via the `blocks.kind` column.
   - Multi-label callers use `set_block_kinds` / `get_block_kind_tags`.

### Deferred (need separate tracks, not foundation work)

3. **Pre-computed kernel candidates** ‚Äî wait until actual kernel
   strategies (surprise_weighted / lineage_first / graph_walk) land;
   the materialised view design depends on what those strategies
   actually filter on.
4. **Vector quantization (PQ)** ‚Äî orthogonal to foundation; ties to
   the embedding backend choice (sqlite-vec vs pgvector) and is a
   storage-layer optimisation, not a v4 surface concern.
5. **HNSW kind index (M=16, efc=200)** ‚Äî depends on ANN backend
   selection; sqlite-vec already exposes HNSW, pgvector route differs.
   Track separately.
6. **Write-time consolidation worker (MemGPT pattern)** ‚Äî needs the
   surprise-weighted clustering primitive, which depends on
   :func:`surprise_retrieval.centroid` being wired through the
   recall log ‚Äî wait for that wiring.

### Also deferred for later

7. **Per-kind global summaries (GraphRAG)** ‚Äî `kind_summary(kind,
   embedding, entity_count)` precomputed on write, fused into DEFAULT
   kernel. Worth doing once the fusion module lands.
8. **Self-editing on recall (MemGPT)** ‚Äî `edit_block_on_recall(block_id,
   delta)` triggered by contradicts_first kernel; integrates with the
   v3.x propose/approve governance flow rather than going around it.
   Defer until v4 governance UX lands.
